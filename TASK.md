# Тестовое задание

**Контекст:**  
Нужно реализовать сервис по продаже билетов на концерт популярной группы. Ожидается, что в момент старта продаж на сайт придут 50 000 человек одновременно.

**Задача:**  
Написать на Node.js (TypeScript) упрощенный API-сервис, который обрабатывает бронирование мест. Ожидается, что выполнение этого задания не займет более 3-4ч.

**Функциональные требования:**  
1. Эндпоинт POST /reserve: Принимает user_id и seat_id.  
2. Бизнес-логика:  
   1. Одно место может забронировать только один пользователь.  
   2. Race Condition: Нужно исключить ситуацию, когда два пользователя одновременно бронируют одно и то же место, и оба получают ответ «Успешно».  
3. Хранение данных: Использовать PostgreSQL или Redis (обосновать свой выбор).  
4. Имитация нагрузки: Реализовать эндпоинт так, чтобы он выдерживал высокую конкурентность (много одновременных запросов к одной строке в БД).
